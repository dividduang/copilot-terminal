# 输入法候选框跳动问题 - 快速参考

## 问题

终端输出时，输入法候选框位置跳动（231px → 252px → 231px）。

## 根本原因

旧方案允许 xterm.js 更新 textarea 位置，然后再恢复：
1. xterm.js 更新位置（231px → 252px）→ 候选框跟随移动
2. 我们恢复位置（252px → 231px）→ 候选框再次移动
3. 用户看到候选框跳动

## 解决方案

**阻止位置变化，而不是变化后再恢复**

```typescript
// 旧方案（错误）
originalUpdate.call(this, dontRecurse);  // ← 允许 xterm.js 更新位置
applyFrozenAnchor(...);                   // ← 然后恢复位置

// 新方案（正确）
if (!frozenAnchor) {
  originalUpdate.call(this, dontRecurse); // ← 首次调用：获取初始位置
  frozenAnchor = readAnchorFromElements(...);
} else {
  // ← 后续调用：跳过原始方法，直接应用冻结锚点
}
applyFrozenAnchor(...);
```

## 关键点

- **输入法候选框位置由操作系统根据 textarea 位置实时计算**
- **任何 textarea 位置变化都会导致候选框立即跟随**
- **即使在一帧内恢复，用户仍能看到跳动**
- **正确做法：完全阻止位置变化**
