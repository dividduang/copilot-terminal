# 输入法候选框跳动问题 - 时序图

## 旧方案（有问题）

```
时间轴 →

用户输入中文
    ↓
[compositionstart] 捕获冻结锚点 (231px)
    ↓
终端输出 (terminal.write)
    ↓
xterm.js 调用 updateCompositionElements
    ↓
执行 originalUpdate.call(this)
    ↓
xterm.js 根据新光标位置更新 textarea
    ↓
textarea 位置变化: 231px → 252px  ← 候选框跟随移动到 252px
    ↓
我们的代码执行 applyFrozenAnchor
    ↓
textarea 位置恢复: 252px → 231px  ← 候选框再次移动到 231px
    ↓
用户看到候选框跳动 ✗
```

## 新方案（已修复）

```
时间轴 →

用户输入中文
    ↓
[compositionstart] 清空冻结锚点
    ↓
xterm.js 调用 updateCompositionElements (首次)
    ↓
执行 originalUpdate.call(this)
    ↓
捕获冻结锚点 (231px)
    ↓
应用冻结锚点
    ↓
终端输出 (terminal.write)
    ↓
xterm.js 调用 updateCompositionElements (第二次)
    ↓
检测到已有冻结锚点
    ↓
跳过 originalUpdate.call(this)  ← 关键：不让 xterm.js 更新位置
    ↓
直接应用冻结锚点 (231px)
    ↓
textarea 位置保持不变: 231px  ← 候选框位置稳定
    ↓
用户看到候选框稳定显示 ✓
```

## 关键差异

| 方案 | 旧方案 | 新方案 |
|------|--------|--------|
| 是否执行原始方法 | 每次都执行 | 首次执行，后续跳过 |
| textarea 位置变化 | 231px → 252px → 231px | 231px → 231px |
| 候选框移动次数 | 2 次（跳动） | 0 次（稳定） |
| 用户体验 | 候选框跳动 ✗ | 候选框稳定 ✓ |

## 代码对比

### 旧方案

```typescript
compositionHelper.updateCompositionElements = function(...) {
  // 问题：每次都执行原始方法
  originalUpdate.call(this, dontRecurse);

  if (!frozenAnchor) {
    frozenAnchor = readAnchorFromElements(...);
  }

  // 问题：变化后再恢复，用户已经看到跳动
  applyFrozenAnchor(textarea, compositionView, frozenAnchor, true);
};
```

### 新方案

```typescript
compositionHelper.updateCompositionElements = function(...) {
  if (!isComposing) {
    originalUpdate.call(this, dontRecurse);
    return;
  }

  if (!frozenAnchor) {
    // 首次：执行原始方法获取初始位置
    originalUpdate.call(this, dontRecurse);
    frozenAnchor = readAnchorFromElements(...);
  } else {
    // 后续：跳过原始方法，避免位置变化
    // 关键：不让 xterm.js 更新位置
  }

  // 应用冻结锚点
  applyFrozenAnchor(textarea, compositionView, frozenAnchor, true);
};
```

## 测试日志对比

### 旧方案日志

```
[IME-FIX] updateCompositionElements 调用 #5
[IME-FIX] updateCompositionElements 原始方法执行后 { afterTop: '252px', changed: true }
[IME-FIX] 应用冻结锚点完成 { finalTop: '231px' }
```
→ 位置变化了，候选框跳动

### 新方案日志

```
[IME-FIX] updateCompositionElements 调用 #2 { hasFrozenAnchor: true }
[IME-FIX] 跳过原始方法执行，直接应用冻结锚点
[IME-FIX] 应用冻结锚点完成 { finalTop: '231px' }
```
→ 位置没有变化，候选框稳定

## 总结

**预防胜于治疗**：与其让位置变化后再恢复，不如从源头上阻止位置变化。
