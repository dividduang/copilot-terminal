# 输入法候选框修复方案 - 暂停渲染

## 问题分析

### 之前方案的失败原因

之前的方案试图通过冻结 `textarea` 和 `compositionView` 的 `style.left/top` 来固定输入法候选框位置，但这个方案失败了。

**根本原因**：输入法候选框的锚点可能不是直接基于 `textarea.style.left/top`，而是基于：
1. 浏览器计算后的实际渲染位置（Computed Style）
2. Canvas 上的光标位置
3. 其他内部机制

因此，即使我们冻结了 `style.left/top`，候选框仍然会跟随终端内容的变化而跳动。

### 新方案的核心思路

**在输入法组合期间暂停终端输出的渲染**，从根源上阻止终端内容变化，从而避免候选框跳动。

## 实现方案

### 1. 修改 `terminalImeFix.ts`

添加 `ImeCompositionState` 类型，用于在组件间共享输入法组合状态：

```typescript
export type ImeCompositionState = {
  isComposing: boolean;
};

export function installStableImeCompositionAnchor(
  terminal: Terminal,
  compositionState: ImeCompositionState
): () => void {
  // ...

  const handleCompositionStart = () => {
    compositionState.isComposing = true;
    // ...
  };

  const handleCompositionEnd = () => {
    compositionState.isComposing = false;
    // ...
  };
}
```

### 2. 修改 `TerminalPane.tsx`

在 `flushOutput` 函数中检查输入法组合状态，如果正在组合则延迟刷新：

```typescript
const compositionStateRef = useRef<ImeCompositionState>({ isComposing: false });

const flushOutput = () => {
  outputFlushFrameRef.current = null;

  const pending = outputBufferRef.current;
  if (!pending || !terminalRef.current) {
    return;
  }

  // 如果正在输入法组合中，延迟刷新
  if (compositionStateRef.current.isComposing) {
    console.log('[TerminalPane] 输入法组合中，延迟刷新输出');
    outputFlushFrameRef.current = requestAnimationFrame(flushOutput);
    return;
  }

  outputBufferRef.current = '';
  terminalRef.current.write(pending);
};
```

## 工作原理

1. **组合开始**：用户按下输入法键，触发 `compositionstart` 事件
   - `compositionState.isComposing` 设置为 `true`

2. **组合期间**：
   - PTY 输出继续到达，数据累积在 `outputBufferRef.current` 中
   - `flushOutput` 被调用，但检测到 `isComposing = true`
   - 输出刷新被延迟，终端内容不更新
   - 输入法候选框保持在原位置，不会跳动

3. **组合结束**：用户选择候选词或按 ESC，触发 `compositionend` 事件
   - `compositionState.isComposing` 设置为 `false`
   - 下次 `flushOutput` 调用时，累积的输出会被一次性刷新到终端

## 优势

1. **从根源解决问题**：不是试图"修复"候选框位置，而是阻止导致候选框跳动的根本原因（终端内容变化）
2. **简单可靠**：不依赖 xterm.js 的内部实现细节
3. **用户体验好**：组合期间终端暂停更新，组合结束后立即恢复，延迟几乎不可察觉

## 潜在问题

1. **输出延迟**：如果用户在输入法组合期间有大量终端输出，这些输出会被延迟到组合结束后才显示
   - **影响评估**：输入法组合通常只持续 1-2 秒，延迟可接受

2. **极端情况**：如果用户长时间保持输入法组合状态（例如打开候选框但不选择），输出会一直被延迟
   - **缓解措施**：`outputBufferRef` 有大小限制（100KB），超过限制会强制刷新

## 测试建议

1. **基本功能测试**：
   - 在终端中输入中文，观察候选框是否稳定
   - 在终端有输出时输入中文，观察候选框是否跳动

2. **边界情况测试**：
   - 在高频输出时输入中文（例如 `ping` 命令运行时）
   - 长时间保持输入法组合状态
   - 快速切换输入法组合状态

3. **对比测试**：
   - 与 Windows Terminal 对比，确认体验一致

## 后续优化

如果这个方案仍然不能完全解决问题，可以考虑：

1. **更激进的暂停策略**：在组合期间完全暂停 xterm.js 的渲染循环
2. **Canvas 冻结**：在组合期间冻结 Canvas 的渲染，不更新任何内容
3. **光标位置锁定**：在组合期间锁定光标位置，不响应任何移动命令
