# 输入法候选框修复方案：禁用渲染循环

## 问题背景

在动画场景下（例如 Claude Code CLI 的读秒动画：1s → 2s → 3s...），xterm.js 的 Canvas 不断重绘，导致输入法候选框位置被抢夺，出现来回闪动的问题。

## 之前尝试的方案（均失败）

1. **修改 textarea 的 style.left/top**：无效，因为候选框位置不是由 textarea 的 style 决定的
2. **暂停终端输出刷新**：无效，因为 Canvas 仍在重绘
3. **使用 MutationObserver 恢复位置**：无效，因为候选框位置不是由 DOM 属性决定的

## 新方案：在组合期间完全禁用 xterm.js 的渲染循环

### 核心思路

在 IME 组合期间（compositionstart → compositionend）：
1. **禁用 xterm.js 的渲染循环**：Hook `_renderService._renderRows` 方法，使其直接返回，不执行任何渲染
2. **延迟输出刷新**：在组合期间，延迟 PTY 输出的刷新，避免触发渲染
3. **组合结束后恢复**：恢复原始的渲染方法和输出刷新

### 实现细节

#### 1. 修改 `terminalImeFix.ts`

```typescript
// 扩展类型定义，包含渲染服务
type RenderServiceLike = {
  dimensions: RenderDimensionsLike;
  _renderRows?: (start: number, end: number) => void;
  _onRender?: { fire: (event: any) => void };
};

// 在 handleCompositionStart 中禁用渲染
const handleCompositionStart = () => {
  // ... 其他代码 ...

  // 关键：禁用 xterm.js 的渲染循环
  const renderService = core?._renderService;
  if (renderService && renderService._renderRows && !originalRenderRows) {
    originalRenderRows = renderService._renderRows;
    renderService._renderRows = function blockedRenderRows(start: number, end: number): void {
      renderBlockCount++;
      debugLog(`渲染被阻止 #${renderBlockCount}`, { start, end });
      // 完全不执行渲染，直接返回
    };
    debugLog('已禁用 xterm.js 渲染循环');
  }
};

// 在 handleCompositionEnd 中恢复渲染
const handleCompositionEnd = () => {
  // ... 其他代码 ...

  // 恢复 xterm.js 的渲染循环
  const renderService = core?._renderService;
  if (renderService && originalRenderRows) {
    renderService._renderRows = originalRenderRows;
    originalRenderRows = null;
    debugLog('已恢复 xterm.js 渲染循环');
  }
};
```

#### 2. 修改 `TerminalPane.tsx`

```typescript
// 导入 IME 修复
import { installStableImeCompositionAnchor, type ImeCompositionState } from '../utils/terminalImeFix';

// 添加 IME 状态 ref
const imeCompositionStateRef = useRef<ImeCompositionState>({ isComposing: false });

// 在终端初始化后安装 IME 修复
const disposeImeFix = installStableImeCompositionAnchor(terminal, imeCompositionStateRef.current);

// 在输出刷新时检查 IME 状态
const flushOutput = () => {
  // ... 其他代码 ...

  // 如果正在组合输入，延迟刷新输出，避免干扰候选框
  if (imeCompositionStateRef.current.isComposing) {
    outputFlushFrameRef.current = requestAnimationFrame(flushOutput);
    return;
  }

  // ... 执行刷新 ...
};

// 在清理函数中调用 disposeImeFix
return () => {
  disposeImeFix();
  // ... 其他清理 ...
};
```

### 预期效果

1. **组合期间**：
   - xterm.js 的渲染循环被完全禁用
   - PTY 输出被延迟刷新
   - Canvas 不会重绘
   - 输入法候选框位置保持稳定

2. **组合结束后**：
   - 渲染循环恢复正常
   - 延迟的输出被刷新
   - 终端显示恢复正常

### 测试方法

1. 启动 Claude Code CLI（或其他有动画输出的应用）
2. 在终端中输入中文，触发输入法候选框
3. 观察候选框是否稳定，不再闪动

### 注意事项

1. **使用自定义 xterm.js**：此方案依赖于访问 xterm.js 的内部 API（`_renderService._renderRows`），需要使用自定义版本的 xterm.js
2. **性能影响**：在组合期间，终端输出会被延迟，但通常组合时间很短（几百毫秒），不会有明显的性能影响
3. **兼容性**：此方案仅在 Windows 上测试，其他平台可能需要调整
