# 输入法候选框修复 - 实施总结（最新版本）

## 问题回顾

在终端中使用中文输入法时，输入法候选框位置不稳定，会随着终端内容刷新而不断闪动和跳动。

**特别是在动画场景下**（例如 Claude Code CLI 的读秒动画：1s → 2s → 3s...），xterm.js 的 Canvas 不断重绘，导致候选框位置被抢夺。

## 解决方案演进

### 方案 1：暂停输出刷新（已失败）

**思路**：在输入法组合期间延迟 PTY 输出的刷新

**结果**：无效，因为 Canvas 仍在重绘（即使没有新输出，动画也会触发重绘）

### 方案 2：禁用渲染循环（当前方案）

**思路**：在输入法组合期间完全禁用 xterm.js 的渲染循环

**核心**：Hook `_renderService._renderRows` 方法，使其在组合期间直接返回，不执行任何渲染

```typescript
  dimensions: RenderDimensionsLike;
  _renderRows?: (start: number, end: number) => void;
  _onRender?: { fire: (event: any) => void };

type TerminalCoreLike = {

  _compositionHelper?: CompositionHelperLike;
  _compositionView?: HTMLElement;
  _syncTextArea?: () => void;
  _renderService?: RenderServiceLike;

```

#### 在组合开始时禁用渲染

```typescript
const handleCompositionStart = () => {
  debugLog('组合开始 (compositionstart)');
  compositionState.isComposing = true;
  frozenAnchor = null;
  updateCallCount = 0;
  mutationCount = 0;
  blockUpdateCount = 0;
  renderBlockCount = 0;

  // 关键：禁用 xterm.js 的渲染循环
  const renderService = core?._renderService;
  if (renderService && renderService._renderRows && !originalRenderRows) {
    originalRenderRows = renderService._renderRows;
    renderService._renderRows = function blockedRenderRows(start: number, end: number): void {
      renderBlockCount++;
      debugLog(`渲染被阻止 #${renderBlockCount}`, { start, end });
      // 完全不执行渲染，直接返回
    };
    debugLog('已禁用 xterm.js 渲染循环');
  }
};
```

#### 在组合结束时恢复渲染

```typescript
const handleCompositionEnd = () => {
  debugLog('组合结束 (compositionend)', {
    updateCallCount,
    mutationCount,
    blockUpdateCount,
    renderBlockCount,
    finalAnchor: frozenAnchor
  });
  compositionState.isComposing = false;
  frozenAnchor = null;

  // 恢复 xterm.js 的渲染循环
  const renderService = core?._renderService;
  if (renderService && originalRenderRows) {
    renderService._renderRows = originalRenderRows;
    originalRenderRows = null;
    debugLog('已恢复 xterm.js 渲染循环');
  }

  scheduleLiveCursorSync();
};
```

## 工作原理

### 问题根源

在动画场景下，xterm.js 的 Canvas 不断重绘，每次重绘都可能触发某些更新，导致输入法候选框位置被抢夺。

### 解决方案

在 IME 组合期间（compositionstart → compositionend）：

1. **禁用渲染循环**：Hook `_renderService._renderRows` 方法，使其直接返回，不执行任何渲染
2. **延迟输出刷新**：在 `flushOutput` 中检查 IME 状态，组合期间延迟刷新
3. **组合结束后恢复**：恢复原始的渲染方法和输出刷新

### 效果

- 组合期间，Canvas 不会重绘
- 输入法候选框位置保持稳定
- 组合结束后，终端显示恢复正常

## 注意事项

1. **使用自定义 xterm.js**：此方案依赖于访问 xterm.js 的内部 API（`_renderService._renderRows`），需要使用自定义版本的 xterm.js
2. **性能影响**：在组合期间，终端输出会被延迟，但通常组合时间很短（几百毫秒），不会有明显的性能影响
3. **兼容性**：此方案仅在 Windows 上测试，其他平台可能需要调整

## 后续优化

如果此方案仍然无法完全解决问题，可以考虑：

1. **使用 CSS 固定 textarea 位置**：在组合期间，使用 `position: fixed` 或其他 CSS 技巧固定 textarea 位置
2. **修改 xterm.js 源码**：直接在 xterm.js 源码中添加"组合期间跳过渲染"的逻辑
3. **替换 textarea**：在组合期间，创建一个新的 textarea 元素，使用固定位置，组合结束后再切换回原 textarea
